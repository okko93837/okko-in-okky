# 오늘의 코디 - 서비스 기획서

## 1. 서비스 개요

### 1.1 컨셉
사용자가 전신 사진 1장을 업로드하면, AI가 의류를 자동으로 분리·분석·정제하여 개인 옷장에 저장하고, 옷장에서 아이템을 조합하면 프리셋 모델이 착용한 미리보기 이미지를 생성하는 웹 서비스.

### 1.2 목적
- 보유한 옷의 코디 조합을 실제로 입어보지 않고 시각적으로 확인
- AI 기반 의류 분석 및 가상 착용 미리보기 제공
- 해커톤 프로토타입 → 추후 코디 추천 서비스로 확장 가능

### 1.3 플랫폼
- **웹앱** (모바일 반응형 UI 고려, 네이티브 앱 미고려)

---

## 2. 사용자 플로우

```
[사용자]
    │
    ├─ 1. 전신 사진 업로드 (1장)
    │
    ├─ 2. 자동 처리 대기
    │     └─ 블러 → 세그멘테이션 → 배경 제거 → 재질 파악 → 제품샷 생성 → 옷장 저장
    │
    ├─ 3. 옷장 확인
    │     └─ 저장된 의류 아이템 (상의 / 하의 / 신발) 목록 확인
    │
    ├─ 4. 코디 조합
    │     ├─ 상의 선택
    │     ├─ 하의 선택
    │     ├─ 신발 선택
    │     └─ 프리셋 모델 선택
    │
    └─ 5. 미리보기 확인
          └─ 프리셋 모델이 선택한 옷을 입은 전신 이미지 확인
```

## 3. 시스템 아키텍처

### 3.1 전체 파이프라인 개요

```
┌─────────────────────────────────────────────────────────────────┐
│                     인입 파이프라인 (옷장 구성)                      │
│                                                                 │
│  사진 업로드 → 얼굴 블러 → SAM → rembg → 재질 파악 → 제품샷 생성 → 저장  │
│  (클라이언트)  (클라이언트)  (API)  (서버)   (API)      (API)     (DB)  │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                   미리보기 파이프라인 (코디 확인)                     │
│                                                                 │
│  아이템 선택 + 프리셋 모델 선택 → 착용 이미지 생성 → 미리보기 출력       │
│       (클라이언트)                    (API)          (클라이언트)     │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 인입 파이프라인 상세

#### Step 1: 사진 업로드
- **입력**: 전신 사진 1장
- **조건**: 상의, 하의, 신발이 모두 보이는 전신 사진
- **처리 위치**: 클라이언트

#### Step 2: 얼굴 블러 (클라이언트)
- **목적**: 프라이버시 보호 - 서버에 얼굴 데이터 미전송
- **기술 스택**:
  - OpenCV.js FaceDetectorYN + YuNet ONNX 모델
  - 모델 파일: `face_detection_yunet_2023mar.onnx` (227KB, OpenCV Zoo 다운로드)
  - Canvas API 픽셀화 (축소 후 `imageSmoothingEnabled = false`로 확대, 블록 크기 8px)
- **함수 인터페이스**:
  - `initFaceDetector()`: OpenCV.js 로드 + YuNet 모델 초기화. 앱 시작 시 1회 호출
  - `mosaicFaces(imageSource)`: 이미지를 받아 얼굴 모자이크 처리된 Canvas 반환
- **주의사항**:
  1. RGBA → BGR 변환 필수: `cv.imread(canvas)`는 RGBA(4채널)를 반환하는데 `FaceDetectorYN.detect()`는 BGR(3채널)만 받음. `cv.cvtColor(src, dst, cv.COLOR_RGBA2BGR)` 변환 후 detect에 넘길 것. 미변환 시 "ERR: undefined" 발생
  2. Canvas를 경유해서 읽을 것: `<img>`를 `cv.imread`에 직접 넘기면 hidden이거나 렌더링 안 된 경우 크기가 0으로 읽힘. Canvas에 `drawImage`로 먼저 그린 뒤 그 Canvas를 `cv.imread`에 전달
  3. OpenCV.js WASM 초기화 대기: 스크립트 로드 후에도 WASM 런타임 초기화가 필요함. `cv.Mat`이 존재하는지 확인 후 진행, 아니면 `cv['onRuntimeInitialized']` 콜백 등록
  4. WASM 메모리 수동 해제: OpenCV.js의 Mat, detector 등은 JS GC가 관리하지 않음. 사용 후 반드시 `.delete()` 호출
  5. 모델 로드는 `fetch` + `FS_createDataFile`: ONNX 파일을 fetch로 받은 뒤 `cv.FS_createDataFile('/', 'yunet.onnx', new Uint8Array(buf), true, false, false)`로 WASM 가상 파일시스템에 마운트. `FaceDetectorYN` 생성자에는 가상 경로 `'/yunet.onnx'`를 전달

#### Step 3: SAM 세그멘테이션 (서버)
- **목적**: 전신 사진에서 상의, 하의, 신발을 개별 이미지로 분리
- **기술**: Replicate SAM3 (`yodagg/sam3-image-seg`)
- **URL**: https://replicate.com/yodagg/sam3-image-seg
- **비용**: ~$0.0017/건 ($1당 약 588회)
- **카테고리 지정**: 상의 / 하의 / 신발로 세그멘테이션 주제 전달
- **출력**: 카테고리별 분리된 이미지 3장

#### Step 4: rembg 배경 제거 (서버)
- **목적**: 세그멘테이션된 각 아이템의 배경 완전 제거
- **기술**: rembg 라이브러리
- **입력**: SAM으로 분리된 의류 이미지 (상의, 하의, 신발 각각)
- **출력**: 투명 배경의 의류 이미지 3장

#### Step 5: 멀티모달 LLM 재질 파악 (서버)
- **목적**: 의류의 재질/소재를 텍스트로 분석
- **입력**: rembg 처리된 의류 이미지
- **출력**: 재질 텍스트 설명 (예: "면 소재 버건디 셔츠", "코듀로이 베이지 와이드 팬츠")
- **용도**: 다음 단계(제품샷 생성)의 프롬프트 입력으로 활용

#### Step 6: gpt-image-1.5 제품샷 생성 (서버)
- **목적**: 모델이 입고 있던 구겨진/비틀린 형태 → 깔끔하게 정돈된 제품샷 이미지로 변환
- **입력**:
  - rembg 처리된 의류 이미지 (이미지)
  - 재질 파악 결과 텍스트 (프롬프트)
- **출력**: 깔끔한 제품샷 스타일 이미지
- **제약 조건 (프롬프트 내 명시)**:
  - 원본 의류의 색상 변경 금지
  - 패턴/무늬 임의 수정 금지
  - 재질감 임의 변경 금지
  - 디자인 요소(단추, 지퍼, 로고 등) 추가/제거 금지

#### Step 7: 옷장 저장
- **저장 데이터**:
  - 생성된 제품샷 이미지
  - 메타데이터: 카테고리(상의/하의/신발), 재질 텍스트, 색상 정보
- **구조**: 사용자별 옷장 → 카테고리별 분류

### 3.3 미리보기 파이프라인 상세

#### Step 1: 아이템 선택
- **옷장 UI**에서 카테고리별 아이템 선택
  - 상의 1개 선택
  - 하의 1개 선택
  - 신발 1개 선택

#### Step 2: 프리셋 모델 선택
- **프리셋 모델**: AI로 사전 생성된 고정 에셋
- **포함 정보**: 체형 + 일관된 포즈
- **특징**: 모든 미리보기에서 동일한 포즈로 일관성 유지

#### Step 3: gpt-image-1.5 착용 이미지 생성
- **입력**:
  - 의류 제품샷 이미지 3장 (옷장에서 선택된 상의/하의/신발)
  - 프리셋 모델 레퍼런스 이미지
- **임의판단 금지 조항** (프롬프트 내 필수 포함):
  - 원본 의류의 색상을 절대 변경하지 않을 것
  - 패턴, 무늬, 프린트를 임의로 수정하지 않을 것
  - 재질감을 다른 소재로 변경하지 않을 것
  - 의류에 없는 디자인 요소를 추가하거나, 있는 요소를 제거하지 않을 것
  - 의류의 핏(슬림, 오버사이즈 등)을 임의로 변경하지 않을 것
- **출력**: 프리셋 모델이 선택한 의류 조합을 착용한 전신 이미지

## 9. 제약 조건 및 고려사항

### 9.1 프라이버시
- 얼굴 데이터는 서버에 절대 전송하지 않음
- 클라이언트에서 블러 처리 완료 후 서버 전송

### 9.2 이미지 생성 품질 관리
- 임의판단 금지 조항을 프롬프트에 반드시 포함
- 원본 의류의 시각적 특성(색상, 패턴, 무늬, 재질감, 핏) 보존이 핵심
- 프리셋 모델의 포즈 일관성 유지

### 9.3 해커톤 제약
- 제한 시간: 4시간
- 플랫폼: 웹앱
- MVP 중심 개발, 확장 기능은 추후 고려
