## 1. 파이프라인 최적화

현재 파이프라인은 모든 단계가 직렬로 실행되어, 아이템 3개 기준 약 2분 이상 소요된다.
병렬화와 구조 개선을 통해 **체감 처리 시간을 절반 이하로 단축**할 수 있다.

### 현재 병목 분석

```
[현재 — 직렬 파이프라인]

SAM 세그멘테이션 (10~20초)
  → 아이템 1: 배경제거(10초) → 재질분석(3초) → 제품샷(12초) → 제품샷 배경제거(10초)
    → 아이템 2: 배경제거 → 재질분석 → 제품샷 → 제품샷 배경제거
      → 아이템 3: 배경제거 → 재질분석 → 제품샷 → 제품샷 배경제거

총 소요: ~20초 + 35초 × 3개 = ~125초
```

### 최적화 1: SAM과 재질 분석 병렬 실행

현재는 SAM → 크롭 → 배경제거 → 재질분석 순서로 아이템별 개별 호출한다.
**모자이크 처리된 전체 사진을 GPT-4o에 바로 넘겨서 SAM과 동시에 재질/색상을 일괄 추출**하면,
세그멘테이션이 끝나는 시점에 이미 재질 데이터가 준비되어 있다.

```
[개선 — 병렬 재질 분석]

┌─ SAM 세그멘테이션 (10~20초)
│
├─ GPT-4o 전체사진 재질 일괄 분석 (3초)  ← 동시 실행
│  "상의: 면/흰색, 하의: 데님/인디고, 신발: 가죽/검정"
│
└→ SAM 결과 + 재질 매칭 → 아이템 처리 시작 (재질 분석 단계 생략)

절감: 아이템당 ~3초 × 3개 = ~9초
```

### 최적화 2: 아이템 병렬 처리

현재 아이템 3개를 순차 처리하고 있다.
API rate limit 범위 내에서 **2~3개 아이템을 동시에 처리**하면 시간이 크게 단축된다.

```
[개선 — 병렬 아이템 처리]

SAM 완료 후:
  ┌─ 아이템 1: 배경제거 → 제품샷 → 제품샷 배경제거
  ├─ 아이템 2: 배경제거 → 제품샷 → 제품샷 배경제거  ← 동시 실행
  └─ 아이템 3: 배경제거 → 제품샷 → 제품샷 배경제거  ← 동시 실행

총 소요: ~20초 + ~35초 = ~55초 (기존 125초 대비 56% 단축)
```

### 최적화 3: 배경제거 모델 프리로딩

@imgly/background-removal ONNX 모델은 첫 호출 시 로딩에 3~5초가 소요된다.
사용자가 **블러 확인 화면에서 "확인" 버튼을 누르기 전에 모델을 미리 로딩**해두면
첫 배경 제거 시 대기 시간을 제거할 수 있다.

```
[개선 — 프리로딩]

블러 확인 화면 진입 시 → 백그라운드에서 @imgly 모델 다운로드 시작
사용자가 확인 → SAM 호출 + 모델 로딩 이미 완료 → 즉시 배경 제거 시작

절감: ~3~5초 (첫 호출 콜드스타트 제거)
```

### 최적화 4: 제품샷 배경 처리 개선

현재는 gpt-image-1으로 제품샷 생성 후, 다시 @imgly로 배경 제거를 한다.
**프롬프트에서 투명/순백 배경을 강제**하거나 **서버사이드 rembg**로 전환하면
클라이언트 배경 제거 단계를 줄일 수 있다.

```
[개선 — 배경 처리 경량화]

방안 A: 프롬프트 강화 → "pure #FFFFFF background" → 후처리 단순화
방안 B: 서버 rembg 전환 → 클라이언트 WASM 대비 2~3배 빠른 처리
방안 C: 배경 제거 생략 → 제품샷 자체가 충분히 깔끔하면 스킵

절감: 아이템당 ~5~10초
```

### 최적화 5: 이미지 해상도 최적화

현재 원본 해상도 그대로 모든 API에 전달하고 있다.
**재질 분석에는 저해상도로 충분**하고, SAM도 리사이즈된 이미지에서 동작 가능하다.

```
재질 분석: 512px 리사이즈 (현재 원본 전달 → 불필요한 토큰 소비)
SAM 세그멘테이션: 1024px 리사이즈 (정확도 유지하면서 전송 시간 단축)
제품샷 생성: 현재 1024×1024 유지 (최종 결과물이므로)

절감: API 응답 시간 ~20~30% 감소 + 비용 절감
```

### 최적화 종합 효과

```
[Before]  SAM(20초) → 아이템×3 순차(35초×3) = ~125초

[After]
  ┌─ SAM (15초, 리사이즈)
  ├─ 재질 일괄 분석 (3초, 병렬)     ← 최적화 1
  ├─ @imgly 프리로딩 (0초, 이미 완료) ← 최적화 3
  └→ 아이템×3 병렬(30초, 1사이클)    ← 최적화 2, 4, 5

  총 소요: ~15초 + ~30초 = ~45초 (기존 대비 ~64% 단축)
```

## 2. AI 코디 추천 엔진

현재는 사용자가 직접 상의/하의/신발을 골라 미리보는 구조이다.
이를 **AI가 옷장 데이터를 분석해서 코디를 제안**하는 방향으로 확장한다.

### 핵심 기능

- **색상 조화 분석**: 이미 저장 중인 `material`과 `color` 메타데이터를 활용하여 색상 이론(보색/유사색/톤온톤) 기반 조합 점수 산출
- **TPO 추천**: "오늘 날씨 + 일정"을 입력하면 옷장에서 적합한 코디를 자동 생성
- **스타일 학습**: 사용자가 선택한 코디 이력을 학습해서 개인 취향 반영

### 확장 가능 근거

현재 파이프라인이 이미 GPT-4o로 재질/색상을 분석하고 있다.
이 메타데이터를 기반으로 추천 로직을 얹는 것은 자연스러운 확장이다.

## 3. 쇼핑몰 연동 — 상품 매칭 & 광고

옷장에 저장된 아이템을 기반으로 **실제 쇼핑몰 상품과 매칭**한다.

### 핵심 기능

- **유사 상품 검색**: 제품샷 이미지 임베딩(CLIP 등)으로 쇼핑몰 DB에서 비슷한 옷 검색
- **코디 완성 추천**: "이 상의에 어울리는 하의"를 쇼핑몰 상품으로 제안
- **가격 비교**: 같은 아이템을 여러 쇼핑몰에서 비교
- **"이 옷 어디서 샀어?" 역검색**: 사진 속 옷과 동일/유사한 상품을 쇼핑몰에서 찾아주기

### 수익 모델

- 쇼핑몰 제휴 커미션 (CPA/CPC)
- 브랜드 광고 노출
- 프리미엄 추천 구독

## 4. 소셜 코디 공유 & 커뮤니티

개인 옷장을 넘어 **사용자 간 코디를 공유하는 소셜 기능**으로 확장한다.

### 핵심 기능

- **코디 카드 공유**: 생성한 착용 이미지를 SNS용 카드로 만들어 공유
- **코디 피드**: 다른 사용자들의 코디를 탐색하고 영감 받기
- **"이 코디 따라하기"**: 다른 사용자의 코디에서 내 옷장의 유사 아이템으로 자동 매칭
